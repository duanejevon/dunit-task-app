# 4. ReorderColumnsUI

**Goal**: Let the user drag-reorder columns within a board, with a visible grip-icon drag handle on each column header so it's clear columns are reorderable.
**Est.**: 1.5h
**Dependencies**: 03.ReorderColumnsBackend

## Steps
- [x] In `src/state/useColumns.ts`, add a `reorderColumns(columnIds: number[])` callback that optimistically reorders local `columns` state, calls `window.taskApi.columns.reorder(columnIds)`, and refreshes/rolls back on error.
- [x] In `src/components/KanbanBoard.tsx`, wrap the columns list in a dnd-kit `SortableContext` (horizontal list strategy, since columns lay out left-to-right), using each column's id as the sortable id. This is a second, separate sortable context from the existing card drag-and-drop `DndContext` — keep card DnD scoped to within `ColumnDropZone`/`CardList` so the two don't conflict.
- [x] Add a small grip/hamburger icon in each `.kanban-column-header`, wired to dnd-kit's `useSortable` `listeners`/`attributes` for that column — positioned before or after the column name, distinct from the existing rename (double-click) and delete (×) controls.
- [x] On drag end, compute the new order with `arrayMove` and call `reorderColumns`.
- [x] Ensure dragging a column by its grip doesn't trigger card drag handlers, and dragging a card doesn't trigger column reorder.

## Definition of Done
- [x] Each column header shows a visible grip icon.
- [x] Dragging a column by its grip to a new position updates the order on screen and persists it (still in new order after app restart).
- [x] Existing card drag-and-drop between/within columns still works unaffected.
- [x] Renaming and deleting columns still work.
