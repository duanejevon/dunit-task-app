# 2. ReorderBoardsUI

**Goal**: Let the user drag-reorder boards in the sidebar list, with a visible grip-icon drag handle on each board item so it's clear the list is reorderable.
**Est.**: 1.5h
**Dependencies**: 01.ReorderBoardsBackend

## Steps
- [x] In `src/state/useBoards.ts`, add a `reorderBoards(boardIds: number[])` callback that optimistically reorders local `boards` state, calls `window.taskApi.boards.reorder(boardIds)`, and refreshes/rolls back on error (mirror the `commitOrder` pattern from `useBoardCards.ts`).
- [x] In `src/components/BoardSwitcher.tsx`, wrap the board `<ul>` in a dnd-kit `DndContext` + `SortableContext` (vertical list strategy), using each board's id as the sortable id.
- [x] Add a small grip/hamburger icon (e.g. "⠿" or an SVG handle) inside each `<li>`, wired to dnd-kit's `useSortable` `listeners`/`attributes`, positioned before the board icon — this is the visible affordance that the row is draggable, separate from the existing click-to-switch and double-click-to-rename interactions.
- [x] On drag end, compute the new order with `arrayMove` and call `reorderBoards`.
- [x] Ensure the grip handle doesn't interfere with the existing `onSwitch` (click) and `startRename` (double-click) handlers on the board name button.

## Definition of Done
- [x] Each board row shows a visible grip icon.
- [x] Dragging a board by its grip to a new position in the list updates the order on screen and persists it (still in new order after app restart).
- [x] Clicking a board name still switches to it; double-clicking still renames it.
