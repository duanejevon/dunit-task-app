# 2. TranslucentSurfaces

**Goal**: Restyle columns/cards/sidebar as semi-transparent panels so the page background is always partially visible through them, built on the Intent 1 tokens. No real background image yet — verify against a temporary placeholder background.
**Est.**: 2h
**Dependencies**: 01

## Steps
- [x] Add translucency tokens (panel alpha values, backdrop-filter blur amount) to the design tokens
- [x] Apply semi-transparent backgrounds + `backdrop-filter` (where supported) to: board-switcher sidebar, kanban columns, kanban cards
- [x] Clamp panel opacity so surfaces never become fully opaque nor fully invisible
- [x] Temporarily set a placeholder body background (gradient or test image) to visually verify translucency before the real background-image feature lands
- [x] Verify text legibility/contrast remains acceptable over the placeholder background

## Definition of Done
- [x] Sidebar, columns, and cards render as translucent surfaces over a placeholder background
- [x] Panels never reach full opacity (background always at least partially visible) nor become too transparent to read
- [x] Build passes; no functional regressions in drag-and-drop or card CRUD

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: Bumped `--surface-0/1/2` alpha (0.16/0.26/0.3) and added `--blur-md: 16px` + `backdrop-filter: blur(...)` to `.board-switcher`, `.kanban-column`, `.kanban-card`. Used the real user-supplied `default.jpg` (not a fake gradient) as a temporary hardcoded `body` background with a `--bg-overlay` white scrim (`linear-gradient` trick) to test legibility against real, busy photo content — confirmed via Playwright screenshot that columns/cards/sidebar read as frosted glass with the image visible through them, and card/column text stayed legible. Marked the body background as provisional in a code comment; Intent 4 replaces it with the dynamic, user-selectable version.
- **Follow-ups**: Contrast was visually fine for this image, but no automated/numeric contrast safeguard exists yet for arbitrary user-chosen images — flagged for Intent 8 (PolishAndReview) to stress-test against a few different image types.
