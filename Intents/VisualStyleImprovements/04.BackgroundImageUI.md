# 4. BackgroundImageUI

**Goal**: Render the chosen background behind the app with a faded overlay, and build the picker UI (gallery + browse button) wired to Intent 3's infra.
**Est.**: 2h
**Dependencies**: 02, 03

## Steps
- [x] Render the active background image full-bleed behind the app shell, with a fixed/semi-transparent overlay so it's always faded regardless of image content
- [x] Build a background picker UI: thumbnails for built-in presets (incl. the default) + a "Browse for image…" button
- [x] Wire picker actions to the Intent 3 IPC (`setBuiltinBackground`, `browseForBackground`)
- [x] Confirm translucent surfaces from Intent 2 layer correctly over the real background image (not just the placeholder)
- [x] Verify the chosen background persists across an app restart

## Definition of Done
- [x] User can choose a background from the gallery or browse for a custom file; it renders faded behind the translucent UI
- [x] Selection persists across restarts
- [x] Legibility holds up across a few different test images (light, dark, busy/high-contrast)

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: Replaced Intent 2's hardcoded body background with a dynamic one: `--app-bg-image` CSS var, set at runtime by a new `useBackground` hook (resolves builtin ids via the bundled `BUILTIN_BACKGROUNDS` registry, custom selections via the `file://` URL from IPC) and applied via `document.documentElement.style.setProperty`. Built `BackgroundPicker.tsx` (gallery thumbnails + "Browse…" button), mounted provisionally at the bottom of the sidebar pending Intent 5's hamburger menu. Hit and fixed a real bug: the global `button:not(...)` accent-color rule had higher CSS specificity than the new `.background-thumb`/`.background-thumb-browse` classes (same issue `.color-swatch` already needed an exclusion for) — added them to the exclusion list. Verified live via Playwright: CSS var resolves to the correct bundled asset URL, thumbnail renders the actual image with a selected-state accent border, smoke tests pass.
- **Follow-ups**: Picker UI lives in the sidebar provisionally — Intent 5 moves it into the hamburger app menu per the original requirement.
