# 3. BackgroundImageInfra

**Goal**: Main-process IPC support for selecting and storing a custom background image, independent of the UI that will consume it.
**Est.**: 2h
**Dependencies**: none (parallel track to 01/02)

## Steps
- [x] Add a small persistence mechanism for app-level settings (a `settings` key/value table in the existing SQLite store) to store the chosen background reference
- [x] Add an Electron main-process handler using `dialog.showOpenDialog` to let the user browse for a local image file
- [x] Copy the selected file into `app.getPath('userData')/backgrounds/` with a generated filename; store its reference in settings
- [x] Expose `getBackground` / `setBuiltinBackground` / `browseForBackground` via the `window.taskApi` IPC bridge
- [x] Bundle the user-supplied default image as an importable asset from `src/assets/backgrounds/` (registry module for future presets)
- [x] Add a smoke test covering: setting a background, restarting the store, confirming the reference persists

## Definition of Done
- [x] IPC round-trip works: set → persisted reference → retrievable on next launch (verified live, including across a full app restart)
- [x] Built-in default image is bundled and selectable via the same API surface
- [x] Smoke test passes
