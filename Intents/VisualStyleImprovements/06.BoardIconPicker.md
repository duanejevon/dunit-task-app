# 6. BoardIconPicker

**Goal**: Per-board icon support — emoji picker plus custom image upload, shown next to the board name.
**Est.**: 2h
**Dependencies**: 03

## Steps
- [x] Add an `icon` column to the boards table (stores either an emoji character or a reference to a copied custom image file)
- [x] Extend board IPC handlers to support setting/updating a board's icon
- [x] Build an emoji picker UI accessible from each board's row in the sidebar
- [x] Add a "custom image" option that reuses the Intent 3 image-copy pattern (copy into `userData/board-icons/`)
- [x] Render the icon (emoji or image) next to the board name in the sidebar

## Definition of Done
- [x] User can assign an emoji or a custom image as a board's icon
- [x] Icon displays next to the board name and persists across restarts
- [x] Existing boards without an icon set still render correctly (sensible default/fallback)

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: Added an `icon` column to `boards` via an `ALTER TABLE` migration (runs on every `createStore` call via a `PRAGMA table_info` check — exercised by every existing smoke-db store creation, so the migration path is implicitly covered). Icon values are either a raw emoji character or a `file://` URL (resolved once at copy time, unlike background's read-time tagged-string resolution — simpler since boards don't need a "builtin id" concept). Added `boards:setIcon`/`boards:browseIcon` IPC, refactored background's copy logic into a shared `pickAndCopyImage(event, subdir)` helper reused by both. Built `BoardIconPicker.tsx`: a per-board popover (same open/outside-click/Escape pattern as `AppMenu`) with a 16-emoji grid + "Custom image…" button. Verified live via Playwright: default fallback icon renders, emoji selection persists via IPC and survives a `boards:list` refetch, smoke tests pass.
- **Follow-ups**: The global `button:not(...)` accent-color exclusion list (`src/index.css`) keeps growing with each new icon-style button class — worth inverting the convention (opt-in accent class instead of opt-out) during Intent 8 polish if it gets unwieldy.
