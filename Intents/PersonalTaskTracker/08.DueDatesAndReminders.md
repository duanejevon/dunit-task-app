# 8. DueDatesAndReminders

**Goal**: Add a due date to a card with a visual indicator and a local reminder notification.
**Est.**: 1.5h
**Dependencies**: 06

## Steps
- [x] Add due date field to the card edit form
- [x] Show a visual indicator on the card (e.g. overdue highlight, date badge)
- [x] Implement local notification scheduling for due reminders
- [x] Handle the app-closed case (e.g. check on next launch) since there's no background process guarantee

## Definition of Done
- [x] Due dates are visible on cards — badge in `CardItem.tsx`, red/bold when overdue
- [x] Reminder notification fires while the app is running or on next launch — `useDueDateReminders` polls every 30s and also runs an immediate check on mount

## Outcome (fill after Iterate)
- **Actual Time**:
- **Result**: `CardForm.tsx` adds a `datetime-local` due-date field (+ Clear button). `CardItem.tsx` shows a due-date badge, styled red/bold once overdue. `electron/db.js#listCardsWithDueDates` + IPC + `window.taskApi.cards.listWithDueDates` expose due cards across *all* boards (not just the active one). `src/state/useDueDateReminders.ts` polls every 30s, schedules a `setTimeout` for upcoming due dates, and fires a web `Notification` immediately for anything already overdue — which is what covers "app was closed when it came due": the immediate check on mount re-notifies on next launch. Wired into `App.tsx` so it runs regardless of which board is open. `npm run smoke:db` extended to verify `due_date` persists and is queryable across a restart — passes. `npm run build` succeeds.
- **Follow-ups**: No "dismiss/acknowledge" state — an overdue card will re-notify on every app launch until its due date is changed or cleared; that's a reasonable nag-until-done default for a personal tracker, but flag if you want a snooze/dismiss control later. Electron's native `Notification` permission is normally auto-granted; if notifications don't appear on your machine, check OS-level notification settings for the app. Same sandbox GUI limitation as prior UI intents — recommend manually testing: set a due date a minute out, confirm the notification fires, and confirm a badge/overdue style shows correctly.
